## 💡 Project Brainstorming (Initial Ideas)

We have shortlisted the following three concepts for our web application. All ideas focus on a **CRUD** (Create, Read, Update, Delete) architecture.

### Option 1: "SkillSwap" – Peer-to-Peer Learning Platform
- **Concept:** A platform where students can exchange skills (e.g., "I teach you Python, you teach me Guitar").
- **Core Features:** User profiles with skill tags, a search/filter system for specific skills, and a booking/request system for sessions.
- **Tech Goal:** Implementing complex filtering and user-to-user interactions.

### Option 2: "EcoTrack" – Personal Sustainability Dashboard
- **Concept:** An app to help users track their daily carbon footprint or plastic usage.
- **Core Features:** Daily activity logs (transport, food, waste), data visualization (charts) of progress over time, and a community board for sharing "green tips."
- **Tech Goal:** Data visualization and handling time-series data.

### Option 3: "StudyBuddy" – Course-Specific Group Finder
- **Concept:** A tool for university students to find or create study groups for specific units.
- **Core Features:** Users can post a "Study Session" with a location/time, others can "Join" the session, and a built-in calendar view.
- **Tech Goal:** Real-time status updates and calendar integration.
